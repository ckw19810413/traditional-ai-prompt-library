# 效率工具 Prompt 集合

本文件包含 25 個精心設計的繁體中文效率工具 prompt。

---

## 一、文件摘要與整理 (5 個 prompt)

### 1. 長文章摘要

```
請幫我摘要以下文章的重點。

文章標題：{標題}
文章內容：
{文章全文或長文本}

要求：
- 生成一個 50 字以內的「一句话摘要」
- 列出 5 個核心重點（每個不超過 30 字）
- 列出 3 個關鍵數據或事實
- 給出「我應該關心這個什麼？」（ relevance ）
- 繁體中文
```

### 2. 會議紀錄生成

```
請根據以下錄音文字生成會議紀錄。

會議主題：{主題}
與會者：{與會人員}
日期：{日期}
錄音文字：
{錄音轉錄文字}

要求：
- 生成結構化會議紀錄
- 包含：會議主題、日期、與會者、議程、討論重點、決議事項、待辦事項
- 待辦事項要明確：誰、做什麼、期限
- 繁體中文
```

### 3. 文件格式轉換

```
請幫我把以下內容轉為 {目標格式：清單/表格/郵件/報告}。

原始內容：
{原始文本}

要求：
- 保持原意不變
- 格式要專業清晰
- 繁體中文
```

### 4. 多語言文件摘要

```
請幫我用繁體中文摘要以下 {源語言} 的文章。

文章標題：{標題}
文章內容：
{文章全文或長文本}

要求：
- 摘要中文化（繁體中文）
- 保留原文的關鍵數據和事實
- 如果原文有專有名詞，在首次出現時保留原文並加註
- 繁體中文
```

### 5. 文件風格改寫

```
請幫我把以下文字的風格改為 {目標風格：正式/輕鬆/學術/簡潔/有說服力}。

原始內容：
{原始文本}

要求：
- 保持原意
- 調整語氣、用詞、句型
- 繁體中文
```

## 二、專案管理 (5 個 prompt)

### 6. 專案計畫生成

```
請幫我制定一個專案計畫。

專案名稱：{專案名}
專案目標：{目標是什麼}
專案範圍：{包含什麼、不包含什麼}
主要交付物：{要交付什麼}
時程：{開始到結束}
資源：{有哪些資源可用}
風險：{已知的風險}

要求：
- 生成一個結構化的專案計畫
- 包含：背景、目標、範圍、時程、資源、風險管理、溝通計畫
- 時程要包含里程碑
- 繁體中文
```

### 7. 專案進度追蹤

```
請幫我生成一個專案進度追蹤報告。

專案名稱：{專案名}
報告期間：{日期範圍}
目前進度：{百分比}
已完成的工作：{完成項目}
進行中的工作：{進行中項目}
遇到的困難：{問題}
下一步：{接下來要做的事}

要求：
- 生成結構化的進度報告
- 包含：整體進度、已完成項目、進行中項目、風險與問題、下一步行動
- 繁體中文
```

### 8. 專案風險分析

```
請幫我分析以下專案的潛在風險。

專案描述：{做什麼}
專案背景：{背景資訊}

要求：
- 列出 5-10 個潛在風險
- 每個風險包含：描述、可能性（高/中/低）、影響（高/中/低）、對策
- 按風險等級排序
- 繁體中文
```

### 9. 專案回顧會議議程

```
請幫我生成一個專案回顧會議的議程。

專案名稱：{專案名}
專案週期：{時間範圍}
團隊規模：{幾個人}
會議時長：{分鐘數}

要求：
- 包含 4 個回顾問題：
  1. 什麼做得好？
  2. 什麼可以改進？
  3. 學到了什麼？
  4. 下一步做什麼？
- 每個問題包含討論時長
- 包含總結環節
- 繁體中文
```

### 10. 待辦事項分類

```
請幫我把以下待辦事項按重要性和急迫性分類。

待辦事項清單：
{待辦事項列表}

要求：
- 使用艾森豪威爾矩陣分類（重要/不重要 × 急迫/不急迫）
- 每個待辦事項要分配一個優先級（P0-P3）
- 給出執行建議順序
- 繁體中文
```

## 三、數據分析 (5 個 prompt)

### 11. 數據趨勢分析

```
請幫我分析以下數據的趨勢。

數據內容：
{數據}

要求：
- 描述整體趨勢
- 識別異常值
- 給出可能的原因假設
- 建議下一步分析方向
- 繁體中文
```

### 12. 報表生成

```
請幫我生成一個數據分析報表。

數據內容：
{數據}

報表目的：{為什麼要看這份報表}
目標讀者：{誰會看這份報表}

要求：
- 生成結構化的報表
- 包含：摘要、關鍵指標、趨勢分析、異常分析、建議
- 繁體中文
```

### 13. 數據解釋

```
請幫我解釋以下數據的含義。

數據內容：
{數據}

背景：{數據的業務背景}

要求：
- 用通俗的語言解釋數據
- 指出數據背後的趨勢或模式
- 給出業務意涵
- 繁體中文
```

### 14. A/B 測試分析

```
請幫我分析 A/B 測試的結果。

測試內容：{測試什麼}
實驗組數據：{實驗組結果}
控制組數據：{控制組結果}
樣本量：{樣本大小}
測試期間：{測試時間}

要求：
- 統計顯著性分析
- 效果大小分析
- 業務影響評估
- 建議（繼續測試/停止/調整）
- 繁體中文
```

### 15. 數據視覺化建議

```
請建議最適合展示以下數據的圖表類型。

數據內容：
{數據}
數據特徵：{時間序列/分類比較/關聯/分佈等}

要求：
- 推薦 3 種圖表類型
- 每種圖表說明：為什麼適合、如何製作、注意什麼
- 繁體中文
```

## 四、知識管理 (5 個 prompt)

### 16. 知識庫分類

```
請幫我把以下知識文章按主題分類。

文章列表：
{文章標題列表}

要求：
- 建立分類體系
- 每篇文章分配一個或幾個分類
- 給出分類的建議名稱
- 繁體中文
```

### 17. 知識連結

```
請幫我找出以下文章之間的關聯。

文章標題：
{文章標題列表}

要求：
- 列出文章之間的關聯
- 推薦可以合併或互鏈的文章
- 給出知識地圖的建議
- 繁體中文
```

### 18. 知識總結

```
請幫我把以下多篇文章的知識匯整成一個摘要。

文章列表：
{文章標題和摘要}

要求：
- 生成一個綜合性摘要
- 提取共同的觀點和差異
- 給出整體知識地圖
- 繁體中文
```

### 19. 知識檢核

```
請幫我檢查以下知識內容是否準確。

知識內容：
{知識內容}

要求：
- 檢查事實正確性
- 指出可能的錯誤或不準確
- 建議補充的內容
- 繁體中文
```

### 20. 知識更新

```
請幫我更新以下知識內容。

原始內容：
{原始內容}
更新日期：{日期}
新的資訊：{新資訊}

要求：
- 更新知識內容
- 保留原有的有效內容
- 標記已更新的部分
- 繁體中文
```

## 五、自動化流程 (5 個 prompt)

### 21. 工作流程自動化建議

```
請幫我建議哪些工作流程可以自動化。

工作場景：{工作場景描述}
使用的工具：{目前使用的工具}
人力投入：{目前花多少時間}

要求：
- 列出 3-5 個可以自動化的流程
- 每個流程包含：自動化方法、工具建議、預期節省時間
- 按實施難度排序
- 繁體中文
```

### 22. Zapier/Make 工作流設計

```
請幫我設計一個自動化工作流。

觸發條件：{什麼情況觸發}
觸發工具：{什麼工具觸發}
要執行的動作：{要做什麼}
目標工具：{要操作什麼工具}

要求：
- 生成 Step-by-step 的工作流設計
- 包含：條件判斷、錯誤處理、通知機制
- 繁體中文
```

### 23. 日誌自動生成

```
請幫我根據今天的活動生成工作日誌。

今天的活動：
{活動描述}

要求：
- 結構化的日誌格式
- 包含：完成事項、進行中事項、問題、明天計畫
- 繁體中文
```

### 24. 報告自動生成

```
請幫我生成週報。

本週活動：
{活動描述}
關鍵指標：
{指標數據}
問題：
{遇到的問題}
下週計畫：
{下週計畫}

要求：
- 結構化的週報格式
- 包含：摘要、完成事項、關鍵指標、問題與解決方案、下週計畫
- 繁體中文
```

### 25. 會議準備清單

```
請幫我生成一個會議準備清單。

會議主題：{主題}
與會者：{與會人員}
會議目的：{要達成什麼}
會議時長：{多長時間}

要求：
- 生成會前、會中、會後的準備清單
- 包含：材料準備、技術檢查、議程設計、記錄要求
- 繁體中文
```
