# 進階應用 Prompt 集合

本文件包含 25 個精心設計的繁體中文進階應用 prompt。

---

## 一、程式碼審查 (5 個 prompt)

### 1. Python 程式碼審查

```
請審查以下 Python 程式碼。

程式碼：
{程式碼}

要求：
- 檢查潛在 bug
- 檢查效能問題
- 檢查安全性問題
- 檢查程式碼風格（PEP 8）
- 給出改進建議（包含修改後程式碼）
- 繁體中文
```

### 2. JavaScript 程式碼審查

```
請審查以下 JavaScript/TypeScript 程式碼。

程式碼：
{程式碼}

要求：
- 檢查潛在 bug 和 edge case
- 檢查 async/await 使用是否正確
- 檢查記憶體洩漏風險
- 檢查安全性問題（XSS、CSRF）
- 給出改進建議
- 繁體中文
```

### 3. SQL 查詢審查

```
請審查以下 SQL 查詢。

查詢：
{SQL 程式碼}
資料庫類型：{PostgreSQL/MySQL/SQLite}
表格結構：
{表格結構}

要求：
- 檢查查詢邏輯是否正確
- 檢查效能問題（缺少索引、不必要的 join）
- 檢查 SQL 注入風險
- 給出優化建議
- 繁體中文
```

### 4. API 設計審查

```
請審查以下 API 設計。

API 端點：
{端點定義}

要求：
- 檢查 RESTful 設計是否正確
- 檢查安全性（認證、授權）
- 檢查錯誤處理
- 檢查文件完整性
- 給出改進建議
- 繁體中文
```

### 5. Git commit message 生成

```
請幫我生成一個符合 Conventional Commits 規範的 commit message。

變更內容：
{git diff 或變更描述}

要求：
- 使用 conventional commits 格式
- 格式：type(scope): description
- type 可以是：feat, fix, docs, style, refactor, test, chore
- 繁體中文
```

## 二、邏輯推理 (5 個 prompt)

### 6. 邏輯題解答

```
請解決以下邏輯題。

題目：{題目}

要求：
- 逐步推理
- 說明每一步的邏輯
- 給出最終答案
- 繁體中文
```

### 7. 決策分析

```
請幫我分析以下決策。

決策情境：{要做的決定}
選項：{可用選項}
考量因素：{要考慮的因素}
限制條件：{限制條件}

要求：
- 列出每個選項的優缺點
- 給出權重分析
- 建議最佳選擇
- 說明假設和風險
- 繁體中文
```

### 8. 因果分析

```
請幫我分析以下事件的因果關係。

事件：{事件描述}
可能的原因：{可能原因}
影響：{產生什麼影響}

要求：
- 區分直接原因和根本原因
- 列出因果關係鏈
- 給出驗證方法
- 繁體中文
```

### 9. 系統思考

```
請幫我用系統思考的方式分析以下問題。

問題：{問題描述}
涉及系統：{系統範圍}

要求：
- 識別系統中的主要要素
- 繪製要素之間的因果循環圖
- 識別反饋迴路
- 給出系統性解決建議
- 繁體中文
```

### 10. 問題解決框架

```
請幫我建立一個問題解決框架。

問題：{問題描述}
已知資訊：{已知資訊}
目標：{要達成的目標}

要求：
- 使用 SCQA 框架（情境、衝突、問題、答案）
- 分解問題成可處理的小問題
- 為每個小問題設計解決策略
- 給出執行順序
- 繁體中文
```

## 三、創意發想 (5 個 prompt)

### 11. 產品功能發想

```
請幫我發想 {產品類型} 的新功能。

目標受眾：{誰會用}
目前功能：{現有功能}
痛點：{受眾的痛點}
競爭對手：{主要競爭對手}

要求：
- 發想 10 個新功能
- 每個功能包含：名稱、描述、價值、開發難度、優先級
- 按創新程度排序
- 繁體中文
```

### 12. 品牌命名

```
請幫我為 {品牌類型} 發想品牌名稱。

品牌定位：{品牌做什麼}
目標受眾：{目標受眾}
品牌調性：{專業/年輕/奢華/親切}
語言偏好：{中文/英文/混合}

要求：
- 提供 20 個名稱選項
- 每個名稱包含：發音、含義、適合原因
- 按適合度排序
- 繁體中文
```

### 13. 行銷活動發想

```
請幫我發想一個行銷活動。

產品：{產品名}
活動目標：{獲客/品牌/銷售}
活動預算：{預算範圍}
目標受眾：{目標客群}
活動時間：{活動期間}

要求：
- 發想 3 個不同的活動概念
- 每個包含：活動名稱、核心概念、執行步驟、預期效果
- 按實施難度排序
- 繁體中文
```

### 14. 故事發想

```
請幫我發想一個故事大綱。

故事類型：{類型}
主題：{核心主題}
目標受眾：{目標讀者}
故事長度：{短篇/中篇/長篇}

要求：
- 提供故事大綱
- 包含：起承轉合
- 主要角色設定
- 衝突與解決
- 繁體中文
```

### 15. 視覺設計概念

```
請幫我生成一個視覺設計概念。

產品/品牌：{做什麼}
品牌調性：{風格}
目標受眾：{目標客群}

要求：
- 描述視覺風格
- 提供配色建議
- 提供排版建議
- 提供視覺元素建議
- 繁體中文
```

## 四、策略規劃 (5 個 prompt)

### 16. 年度策略規劃

```
請幫我制定年度策略規劃。

公司：{公司描述}
當前狀況：{目前狀況}
目標年份：{目標年份}

要求：
- 使用 OKR 框架
- 制定 3-5 個年度目標
- 每個目標包含 3-5 個關鍵結果
- 按季度劃分
- 繁體中文
```

### 17. 商業模式畫布

```
請幫我填寫商業模式畫布。

公司/產品：{公司或產品描述}

要求：
- 填寫商業模式畫布的 9 個區塊：
  1. 客戶細分
  2. 價值主張
  3. 渠道通路
  4. 客戶關係
  5. 收入來源
  6. 關鍵資源
  7. 關鍵活動
  8. 關鍵合作夥伴
  9. 成本結構
- 繁體中文
```

### 18. 市場進入策略

```
請幫我制定市場進入策略。

產品：{產品描述}
目標市場：{目標市場}
競爭格局：{競爭對手}
資源限制：{可用的資源}

要求：
- 使用 STP 框架（細分、目標、定位）
- 制定進入策略
- 規劃 6 個月行動計畫
- 繁體中文
```

### 19. 成長策略

```
請幫我制定成長策略。

公司：{公司描述}
目前規模：{目前規模}
目標：{成長目標}
資源：{可用資源}

要求：
- 使用安索夫矩陣分析成長機會
- 給出 3 個成長策略
- 每個策略包含：執行步驟、資源需求、預期效果、風險
- 繁體中文
```

### 20. 轉型策略

```
請幫我制定轉型策略。

公司現狀：{目前狀態}
目標狀態：{期望狀態}
轉型原因：{為什麼要轉型}
時間框架：{多長時間}

要求：
- 使用 Kotter 8 步變革模型
- 規劃轉型路徑
- 識別潛在阻力
- 給出溝通策略
- 繁體中文
```
